## Methodology: what changed

Round 11 introduces **autonomous scoring** -- a fundamentally different test from previous rounds.

### Previous rounds (R1-R10): Guided scoring

In rounds 1-10, the scoring agent had access to the full conversation context, including golden standards and iterative feedback. This measured "can the agent match a known answer" -- useful for pipeline validation but not representative of real customer experience.

### Round 11: Autonomous scoring

Each library's theme was generated by an **independent subagent** that:
- Had access ONLY to: extraction rules (`theming-agent.mdc`, `theming-reference.md`, `DESIGN-UNIVERSAL-RULES.md`) and the library's `figma-tokens-N.json`
- Had NO access to golden standards, previous scoring results, or conversation context
- Used a fast model (simulating a typical Cursor agent session)
- Generated a complete theme from scratch in a single pass

This tests what a **real customer** would experience: the agent reads the Figma tokens and the rules, then independently generates a theme.

## Key findings

### 1. Autonomous scoring reveals the real gap: 74.3% vs 100%

The 25.7pp drop from guided to autonomous scoring quantifies the gap between "agent with known answers" and "agent in the wild." This is the true measure of agent quality.

### 2. Top failure categories

| Failure type | Libraries affected | Impact |
|-------------|-------------------|--------|
| **Text role misassignment** | 1, 2, 3, 5 | Agent trusts extraction text classification without validating semantic ordering |
| **Chart color ordering** | 2, 3, 4, 5 | Extraction order != design-intent order; agent lacks curatorial judgment |
| **Chromatic cardBg leaking into derivation** | 3 | Misclassified background produces brownish derived text |
| **Shadow selection ambiguity** | 2, 4 | Deterministic rules don't always pick the right shadow |

### 3. Text is the weakest category

Average text score: **56.5%** across all 5 libraries. The agent frequently:
- Uses extraction values verbatim without checking if they satisfy ordering rules
- Fails to recognize when extraction misclassifies text roles (neutral vs subtle vs muted)
- Cannot infer "uniform text" designs (Library 2: all roles use #212529)

### 4. Charts suffer from ordering, not color selection

The agent finds the right colors but often in the wrong order. Extraction provides frequency-based ordering, but golden standards expect brand-priority ordering.

## Recommendations for improvement

### Priority 1: Text role validation rule

Add a post-processing step to the agent rules that validates extracted text tokens against the ordering rules BEFORE using them. If the extraction assigns #666666 to "neutral" when it should be "subtle" (because subtle must be lighter than neutral), the agent should re-classify.

### Priority 2: Chart color intent ordering

The agent needs a heuristic for brand-priority ordering beyond pure extraction frequency. For example: the first chart color should be the one closest to the identified primary brand color.

### Priority 3: Chromatic background rejection cascade

When the agent discards a chromatic background candidate (e.g., #f79e1b), it should also re-derive any text tokens that were computed using that background as an anchor.

### Priority 4: Shadow selection rule refinement

The current "smallest blur >= 10px" fallback picks overly subtle shadows. Consider preferring shadows in the 20-60px range (common for card elevation) before falling back.

---

## Conclusion

Round 11 autonomous scoring establishes a **realistic baseline of 74.3%** for agent quality. This is significantly more honest than the 100% guided score and reveals concrete areas for improvement. The three most impactful fixes (text role validation, chart ordering heuristics, chromatic cascade) could realistically bring autonomous scores to the 85-90% range.
